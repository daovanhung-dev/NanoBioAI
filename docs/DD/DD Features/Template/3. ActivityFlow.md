## 3. Luồng xử lý chi tiết

### Bước 1. UI khởi tạo chức năng
- **UI gọi:** `{{ui_entry_point}}`
- **Mục đích:** mở màn hình / load state / chuẩn bị dữ liệu ban đầu
- **Hành động cụ thể:**
  1. Widget được build.
  2. Provider được đọc bằng `ref.watch` / `ref.read`.
  3. Trigger action load data nếu đây là màn hình đầu tiên.
- **Kết quả mong đợi:** UI chuyển sang trạng thái loading.

### Bước 2. Xác định nguồn dữ liệu
- **Nguồn ưu tiên:** {{data_priority}}
- **Logic chọn nguồn:**
  - Nếu có dữ liệu local hợp lệ → đọc từ SQLite
  - Nếu dữ liệu local thiếu / hết hạn → gọi remote
  - Nếu cần realtime → subscribe để nghe cập nhật
- **Điểm cần lưu ý:** luôn ghi rõ source nào là source of truth theo từng case.

### Bước 3. Lấy dữ liệu từ SQLite
- **Tầng xử lý:** datasource local
- **Function:** `{{sqlite_function_name}}`
- **Input:**
  - `{{sqlite_input_1}}`
  - `{{sqlite_input_2}}`
- **Xử lý chi tiết:**
  1. Mở database / lấy instance
  2. Build query theo điều kiện nghiệp vụ
  3. Thực thi query
  4. Map raw row → model
  5. Validate dữ liệu trước khi trả về
- **Output:**
  - List / object model
  - Empty list nếu không có dữ liệu
- **Error handling:**
  - Database chưa mở
  - Query lỗi
  - Parse model lỗi

### Bước 4. Nếu cần, lấy dữ liệu từ Supabase
- **Tầng xử lý:** datasource remote
- **Function:** `{{supabase_function_name}}`
- **Input:** `{{supabase_input}}`
- **Xử lý chi tiết:**
  1. Gửi request / query tới Supabase
  2. Nhận response
  3. Parse dữ liệu
  4. Map sang model domain
  5. Nếu thành công thì cập nhật SQLite cache
- **Output:** `{{supabase_output}}`
- **Error handling:**
  - Mất mạng
  - Unauthorized
  - Data format không đúng
  - Server error

### Bước 5. Đồng bộ cache local
- **Mục đích:** đảm bảo UI load nhanh và dữ liệu dùng lại được
- **Hành động:**
  1. Xoá cache cũ nếu cần
  2. Insert / update dữ liệu mới vào SQLite
  3. Commit transaction
  4. Emit state mới cho UI
- **Quy tắc đồng bộ:**
  - Dữ liệu remote mới hơn thì ưu tiên ghi đè
  - Nếu local chỉ là cache, không để UI phụ thuộc hoàn toàn vào local khi dữ liệu stale

### Bước 6. Xử lý realtime
- **Mục đích:** cập nhật UI ngay khi dữ liệu thay đổi
- **Hành động:**
  1. Subscribe channel
  2. Lắng nghe event create / update / delete
  3. Khi có event, validate payload
  4. Cập nhật state + cache local
  5. Rebuild UI
- **Cần mô tả rõ:**
  - Event nào update cái gì
  - Event nào chỉ refresh một phần
  - Event nào cần reload toàn bộ

### Bước 7. UI render dữ liệu
- **Khi có dữ liệu:** hiển thị `{{success_ui}}`
- **Khi rỗng:** hiển thị `{{empty_ui}}`
- **Khi lỗi:** hiển thị `{{error_ui}}`
- **Khi loading:** hiển thị `{{loading_ui}}`
