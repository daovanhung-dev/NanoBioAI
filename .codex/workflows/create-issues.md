# Workflow - Create Issues

Use when the user provides findings or asks to turn bugs into issue docs.

## Required Context

- `.codex/AGENTS.md`
- `.codex/DOCS_WORKFLOW.md`
- `.codex/ISSUE_TODO_WORKFLOW.md`
- Evidence supplied by the user or discovered through targeted reads.

## Rules

- Do not fix code.
- Do not create todos unless separately requested.
- Each issue needs severity, expected/actual, reproduction or verification, evidence, impact, and suggested fix direction.

## Completion

- Create issue docs with the next `NNN` in each issue folder.
- Create worklog and refresh `.codex/history/`.
